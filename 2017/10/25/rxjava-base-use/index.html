RxJava操作符

创建操作符

  • create

提供一个接口，它可以将响应式和回调式桥接起来。

  • just

返回一个可观察的结果，它发出一个单一的条目，然后就完成了。

  • fromArray

将一个数组转换成一个可以在数组中释放条目的ObservableSource。

  • interval

创建一个按固定时间间隔发射整数序列的Observable，相当于定时器。

  • range

创建发射指定范围的整数序列的Observable，可以替代for循环，发射一个范围内的有序整数序列。

变换操作符

  • map

通过指定一个Func对象，将Observable转换为一个新的Observable对象并发射，观察者将收到新的Observable处理。

  • flatMap和cast

将Observable发射的数据集合变换为Observable集合，然后将这些Observable发射的数据平坦化地放进一个单独的Observable。

  • concatMap

concatMap操作符功能与flatMap操作符一致；不过，concatMap解决了flatMap交叉问题，提供了一种能够把发射的值连续在一起的函数，而不是合并它们。

  • flatMapIterable

flatMapIterable操作符可以将数据包装成Iterable，在Iterable中可以对数据进行处理。

  • groupBy

groupBy操作符用于分组元素，将源Observable变换成一个发射Observables的新Observable。

过滤操作符

  • filter

filter操作符是对源Observable产生的结果自定义规则进行过滤，只有满足条件的结果才会提交给订阅者。

  • elementAt

elementAt操作符用来返回指定位置的数据。

  • distinct

distinct操作符用来去重，其只允许还没有发射过的数据项通过。和它类似的还有distinctUntilChanged操作符，它用来去掉连续重复的数据。

  • skip和take

skip操作符将源Observable发射的数据过滤掉前n项；而take操作符则只取前n项。另外还有skipLast和takeLast操作符，则是从后面进行过滤操作。

  • ignoreElements

ignoreElements操作符忽略所有源Observable产生的结果，只把Observable的onCompleted和onError事件通知给订阅者。

  • throttleFirst

throttleFirst操作符会定期发射这个时间段里源Observable发射的第一条数据，throttleFirst操作符默认在computation调度器上执行。

组合操作符

  • startWith

startWith操作符会在源Observavle发射的数据前面插上一些数据。

  • merge

merge操作符将多个Observable合并到一个Observable中进行发射，merge可能会让合并的Observable发射的数据交错。

  • concat

将多个Observable发射的数据进行合并发射。concat严格按照顺序发射数据，前一个Observable没发射完成是不会发射后一个Observable的数据的。

  • zip

zip操作符合并两个或多个Observable发射出的数据项，根据指定的函数变换它们，并发射一个新值。

  • combineLatest

当两个Observable中的任何一个发射数据时，使用一个函数结合每个Observable发射的最近数据项，并且基于这个函数的结果发射数据。combineLatest操作符和zip有些类似。只不过zip操作符作用于最近未打包的两个Observable，只有当原始的Observable中的每一个都发射了一条数据时zip才发射数据；而combineLatest操作符作用于最近发射的数据项，在原始Observable中的任意一个发射了数据时发射一条数据。

辅助操作符

  • delay

delay操作符让原始Observable在发射每项数据之前都暂停一段指定的时间段。

  • do

do系列操作符就是为原始Observable的生命周期事件注册一个回调，当Observable的某个事件发生时就会调用这些回调。

  • subscribeOn和observeOn

subscribeOn操作符用于指定Observable自身在哪个线程上运行。如果Observable需要执行耗时操作，一般在子线程进行操作。observeOn用于指定Observer所运行的线程，也就是发射出的数据在哪个线程上使用。一般指定在主线程中执行，这样就可以修改UI了。

  • timeout

timeout操作符会在超时时切换到使用一个指定的备用Observable。

错误处理操作符

  • catch

catch操作符拦截原始Observable的onError通知，将它替换为其他数据项或数据序列，让产生的Observable能够正常终止或者根本不终止。

  • retry

retry操作符不会将原始Observable的onError通知传递给观察者，它会订阅这个Observable，再给它一次机会无错误地完成其数据序列。

条件操作符合布尔操作符

  • all

all操作符根据一个函数对源Observable发射的所有数据进行判断，最终返回的结果就是这个判断结果。

  • contains

contains操作符用来判断源Observable所发射的数据是否包含某一个数据。

  • isEmpty

isEmpty操作符用来判断源Observable是否发射过数据。

  • amb

amb操作符对于给定两个或多个Observable，它只发射首先发射数据或通知的那个Observable的所有数据。

  • defaultIfEmpty

defaultIfEmpty操作符用来发射原始Observable数据。

转换操作符

  • toList

toList操作符将发射多项数据且为每一项数据调用onNext方法的Observable发射的多项数据组合成一个List，然后调用onNext方法传递整个链表。

  • toSortedList

toSortedList操作符与toList类似，区别是，会产生有序的列表，默认是自然排序。

  • toMap

toMap操作符收集原始发射的所有数据项到一个Map，然后发射这个Map。

RxJava的线程控制

内置的Scheduler

  • Schedulers.io()

I/O操作(读写文件、读写数据库、网络信息交互等)所使用的Scheduler

  • Schedulers.newThread()

总是启用新线程

  • Schedulers.computation()

计算所使用的Scheduler

  • Schedulers.single()

返回一个默认的、共享的、单线程支持的调度器实例，用于在相同的后台线程上执行连续的执行

  • Schedulers.trampoline()

当我们想在当前线程执行一个任务时，并不是立即执行，可以用将trampoline加入队列

Android中使用AndroidSchedulers.mainThread()指定操作在主线程中。

控制线程

RxJava中使用subscribeOn()和observeOn控制线程。
